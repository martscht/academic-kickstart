Fragestellung

In diesem Projekt möchten wir das Wissen, was du in den letzten Projekten erlernt hast, verbinden. Wir werden es dennoch so aufbereiten, dass du dieses Projekt bearbeiten kannst, auch wenn du noch nicht an den anderen Projekten gearbeitet hast. So worum geht es hier nun? Wir wollen dir das Web-Scraping zeigen, in dem wir über eine API (Application Programming Interface), Daten herunterladen und diese dann grafisch, wie bei ShinyR aufarbeiten. Eine API ist eine Schnittstelle zwischen Computern. Dies lässt sich zur UI (User Interface) abgrenzen, da es dabei um die Schnittstelle zwischen dem Menschen und dem Computer geht. Die API stellt somit eine Schnittstelle zwischen verschiedenen Softwares dar. Das können bestimmte Dienste sein, die ein Unternehmen zur Verfügung stellt und andere über die API nutzten können, ohne direkt auf den Programmiercode zugreifen zu müssen. Ein anwendungsbezogenes Beispiel wäre dafür ein Online-Store, wo du dich oft über Google anmelden kannst. Dieses Anmeldeverfahren ist über eine von Google bereitgestellte API nutzbar. Außerdem kannst du, wenn du verschiedene Dinge im Online-Store kaufen willst über PayPal bezahlen und nutzt somit die von PayPal bereitgestellte API als Zahlungsverfahren. APIs sind daher heutzutage ein großer Bestandteil des Internets geworden und sind daher wichtig zu kennen.

Zielsetzung

In diesem Projekt wirst du über eine API, Daten in R Studio importieren, um diese grafisch und interaktiv darzustellen. Dafür werden wir die Daten von der WHO (World Health Organisation) verwenden, da diese frei zugänglich sind. Man kann auch von Twitter, Google, YouTube oder anderen Anbietern Daten herunterladen. Dafür ist es jedoch meist erst notwendig sich als Developer zu registrieren, was in unserem Fall zu komplex wird. Wenn du sowas aber mal machen willst, bekommst du einen sogenannten key den du beim Herunterladen der Daten angeben musst. Somit kann das Unternehmen nachvollziehen, wer was mit ihrer Software gemacht hat und somit Missbrauch vorbeugen bzw. nachvollziehen. Für die Daten der WHO ist dies glücklicherweise nicht notwendig. Du findest aber auch viele andere frei zugängliche Daten, wenn du mal nach Open API suchst. Für die grafische Aufbereitung werden wir eine User Interface aufbauen in der man interaktive Plots erschaffen kann, wie bei ShinyR, nur das wir hier das Packet Plotly kennenlernen werden.

Vorbereitung

Am besten schaust du dir zunächst die zugehörige Seite von der WHO hier an. Hier gibt es außerdem eine ausführliche Erklärung, wie du die passenden Links gestalten kannst. Es gibt verschiedene Variablen, wie beispielsweise den Drogenmissbrauch in den verschiedenen Altersgruppen, die Lebenserwartung, Untergewicht bei Kindern unter 5 Jahren oder auch die Anzahl der Toten durch Malaria. Den Link kannst du auch in Excel oder anderen Programmen unter Daten -> Daten abrufen -> Aus anderen Quellen -> Aus OData Datenfeed einlesen. Wir werden das Ganze jedoch R-intern lösen. Als kleine Übung, wie man mit APIs umgeht kannst du dir dieses Tutorial anschauen. Dabei werden Daten über die ISS heruntergeladen. Dies funktioniert über den GET Befehl aus dem httr-Packet. Die heruntergeladenen Daten liegen im JSON Format vor. JSON sind Objekte von JavaScript, eine andere Programmiersprache. Diese Objekte dienen dem einfachen Austausch von Daten zwischen Anwendungen. Du kannst dich gerne näher damit auseinander setzten, wobei das an dieser Stelle für das Projekt nicht notwendig ist. Die JSON Daten lassen sich im zweiten Schritt einfach mit dem jsonlite-Packet umwandeln. Dies funktioniert auch mit den Daten von der WHO, obwohl es sich dabei um OData und keine JSON-files handelt.
