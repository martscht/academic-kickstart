Dieses Projekt befasst sich mit dem Thema Glücksspiel. Zu Glücksspielen zählen Glücksspielautomaten (z.B. der “Einarmige Bandit”), Roulette, Kartenspiele (z.B. Blackjack und Poker), Würfelspiele, Sportwetten, Lotto, Rubbellose und Bingo sowie im weiteren Sinne auch die Börsenspekulation. Wir möchten uns jedoch nur auf das Roulette konzentrieren. Roulette ist ein weltweit verbreitetes, traditionelles Glücksspiel, bei dem es darum geht, auf bestimmte Zahlen oder Eigenschaften von Zahlen zu setzen, die durch den zufälligen Lauf einer Kugel in einem Kessel bestimmt werden (Klicke hier für weitere Informationen). Ziel ist in diesem Projekt, dass du dir am Ende den Weg ins Kasino sparen kannst, weil du dir in R ein Roulette nachgebaut hast!

Zielsetzung

R-intern geht es in diesem Projekt um den Umgang mit Funktionen, mit wenn-dann Bedingungen und das Erzeugen von Schleifen mit for, while und repeat. Eine Übersicht über derartige Funktionen kannst du hier auf der zweiten Seite unter dem Reiter “Programmierung” finden. Das Ziel ist, mithilfe dieser Funktionen ein Roulette-Spiel zu rekonstruieren und damit verschiedene Situation nachzustellen. Letztlich sollst du dich dabei mit dem Gebrauch dieser Funktionen vertraut machen und verschiedene/kombinierte Verwendungsmöglichkeiten kennenlernen.

Beispiel zur Verwendung von Funktionen

Im Folgenden bekommst du ein kleines Beispiel zur Verwendung von Funktionen. Stell dir vor, du spielst mit ein paar Freunden Monopoly und sitzt im Gefängnis fest. Um aus dem Gefängnis zu kommen ist es notwendig, einen Pasch zu würfeln (zwei Würfel - zwei gleiche Augenzahlen). Als erstes möchten wir schauen, bei wie vielen von 50 Würfen ein günstiges Ergebnis, also ein Pasch, herauskommt.

result1 <- NULL
result2 <- NULL
meaning <- NULL
consequence <- NULL

Zuerst erstellt man dazu die Objekte, in denen man im Folgenden die Werte aus der Schleife speichern möchte. result1 sehen wir dafür vor, jeweils die Augenzahl des ersten Würfels zu speichern; result2 soll das gleiche für den zweiten Würfel tun. Die beiden Objekte meaning und consequence sollen jeweils die beiden Augenzahlen kombinieren und zum Einen die Beudeutung, also “Pasch” oder “kein Pasch”, und zum Anderen die daraus folgende Implikation für das Spiel “Du bist frei!” oder “Bleib im Gefängnis!” ausgeben. Die Funktion, die das umsetzt, sieht folgendermaßen aus:

for (i in 1:50) {         # Wie lang soll die Schleife sein? - 50 Wiederholungen
  die1 <- sample(1:6, 1)  # Zufallsziehung von Würfel 1
  result1[i] <- die1      # Das Ergebnis von Würfel 1 wird jeweils in einem neuen Eintrag (Eintrag "i" für die i-te Runde/den i-ten Durchlauf) im Objekt "result1" gespeichert.
  die2 <- sample(1:6, 1)  # Zufallsziehung von Würfel 2
  result2[i] <- die2      # Abspeicherung des Ergebnisses von Würfel 2
  if (die1 == die2) {     # Wenn die beiden Würfel die gleiche Augenzahl haben, dann soll in dem Objekt "meaning" abgespeichert werden, dass ein Pasch gewürfelt wurde.
    meaning[i] <- "Pasch"
  } else {                # Ansonsten soll im Objekt "meaning" abgespeichert werden, dass kein Pasch gewürfelt wurde.
    meaning[i] <- "kein Pasch"
  }
  if (die1 == die2) {     # Wenn die gleiche Augenzahl gewürfelt wurde, dann soll in dem Objekt "consequence" die Anweisung "Du bist frei!" ausgegeben werden.
    consequence[i] <- "Du bist frei!"
  } else {                # Ansonsten soll dort die Anweisung "Bleib im Gefängnis." stehen.
    consequence[i] <- "Bleib im Gefängnis."
  }
}
Monopoly_Prison <- data.frame(result1, result2, meaning, consequence) # Datensatz aus den drei Objekten erstellen
View(Monopoly_Prison)     # Datensatz anschauen

Mit diesem Code “würfelt” man also 50 Mal, speichert die Augenzahlen jeweils ab und fügt Bedeutung (meaning) und daraus folgende Konsequenz für das Spiel (consequence) hinzu. Am Ende liegen in allen 4 Variablen, result1, result2, meaning und consequence, 50 zusammengehörige Ausprägungen vor. Diese kann man dann in einen Datensatz zusammenfügen und erhält folgende Tabelle:

5 1 kein Pasch Bleib im Gefängnis.
6 1 kein Pasch Bleib im Gefängnis.

6 3 kein Pasch Bleib im Gefängnis.

Desweiteren kann man überprüfen, wie lange es dauert, bis man einen Pasch würfelt. Dazu benutzt man folgenden Code:

die3 <- 0    # Würfel 1
die4 <- 1    # Würfel 2
m <- 0       # Anzahl Würfe

Zunächst muss man wiederum 3 Objekte erstellen, die man im Folgenden benutzen möchte. die3 und die4 sollen jeweils die Augenzahl eines Würfels abspeichern. Das wichtige dabei ist, dass man diesen beiden Objekten ungleiche Zahlen zuordnet, dann ansonsten ist die Bedingung der Funktion direkt erfüllt.

m soll dann die Anzahl der Durchgänge zählen, die man bis zum Erfolg (“Pasch”) benötigt. Diese Variable sollte (logischerweise) bei Null beginnen. Warum wird auch in der folgenden Funktion deutlich:

while (die3 != die4) {      # Bedingung
  m <- m + 1                # zählt die Durchgänge - pro Durchlauf +1
  die3 <- sample(1:6, 1)    # Würfel 1
  die4 <- sample(1:6, 1)    # Würfel 2
  print(c(die3, die4))      # Die beiden Augenzahlen werden pro Wurf ausgegeben, um die Ergebnisse nachvollziehen zu können.
}

## [1] 3 6
## [1] 6 3
## [1] 3 2
## [1] 3 1
## [1] 1 4
## [1] 1 1

Jetzt kann man an der Variable m ablesen, wie viele Versuche man bis zu einem Pasch gebraucht hat. Denn angefangen bei 0 wurde m in jedem Durchlauf um 1 erhöht. In diesem Fall hat es erst im siebten Versuch geklappt.
