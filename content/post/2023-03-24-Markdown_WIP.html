Im Folgenden Tutorial wird euch allgemeines Markdown nähergebracht, mit dem rein textbasierte Arbeiten etwas dynamischer und visuell ansprechender gestaltet werden können. Dazu gibt es ein relevantes Anwendungsbeispiel das näherbringen soll wie sich Markdown auf die eigentliche wissenschaftliche Arbeit anwenden lassen kann.

Was ist Markdown?

Markdown ist im allgemeineren eine Auszeichnungssprache, die in fast jeder textbasierten Anwendung in relativ ähnlicher Form vorkommt, um spezifische Textdarstellungen und Inhalte in Texten zu ermöglichen. Es erinnert dabei mehr an eine Codingsprache wie R als an einen schön aufbereiteter Editor wie Word. Ein Beispiel dafür ist die Inline Formatierung. Mit ihr kann man beispielsweise mit **Text** Text fett schreiben oder auch per ~~Text~~ Text durchstreichen. Dies kann man meist in vergleichbarer Form auch in einem Messenger der Wahl replizieren. In WhatsApp lässt sich Text zum Beispiel mit ~Text~ durchstreichen. Man kann auch Links einbinden, indem man den gewollten Textteil in [] Klammern einschließt und den entsprechenden Link in () Klammern setzt. Man kann auch intern auf die jeweiligen Abschnitte verlinken, indem man diese mit einem Link versieht, dazu mehr unter interne Links Markdown-Dateien enden auf .md und können auch innerhalb von RStudio bearbeitet werden. Diese können direkt in RStudio erstellt werden, dafür muss man auf “File”, dann “New File” gehen und weiter unten “Markdown File” auswählen.

Wofür Markdown?

Ein möglicher Einsatz der Formatierung von Markdown und später auch RMarkdown kann sinnvoll sein, wenn man Informationen, etwa wissenschaftliche Forschung oder sonstige studentische Arbeiten auf Websiten und PDFs mit einer Reihe zusätzlicher Funktionen darstellen möchte. Der große Gewinn gegenüber anderen Textformatierungs- und Textverarbeitungsprogrammen wie Word ist das Markdown universell kompilierbar, kostenlos und offen zugänglich ist, womit weder Schreiber noch Leser je an eine spezifische Software und den damit verbundenen Kosten oder der Instandhaltung dieser Software gebunden ist.

Innerhalb dieser Tutorial-Serie werden wir Markdown und Erweiterungen dessen Nutzen, um einen Mini-Bericht zu einer kleinen Auswertung anzufertigen.

Textbestandteile

Es gibt in Markdown auch typische Bestandteile zur visuellen Trennung des Text, wie man sie aus anderen Dokumentbearbeitungsprogrammen auch gewohnt ist. Darunter fallen Zeilenumbrüche und neue Paragraphen. Diese beiden werden auch, wenn man Markdown nutzt um eine Website zu schreiben, zu den entsprechenden HTML-Gegenstücken kompiliert, <p></p> für Paragraphen, <br> für Zeilenumbrüche.

Ein neuer Paragraph wird erstellt, indem man eine Zeile oberhalb und unterhalb des Text freilässt.

Ein Zeilenumbruch wird in Markdown erstellt, indem man zwei Leerzeichen an das Ende einer Zeile setzt und dann per Enter eine neue Zeile beginnt. In dem Fall von Markdown per RStudio braucht man auch keine Leerstellen, aber diese sind insgesamt typisch für allgemeines Markdown und fallen nicht auf, auch wenn sie überschüssig sind.

Heading

Ein typischer Bestandteil eines Dokuments sind die Headings, diese werden in Markdown mit Rauten entsprechend des Heading-Level vor dem Wort erstellt.

Code Level
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
##### Heading 5
###### Heading 6

Es ist wichtig, dass die Headings am Anfang der Zeile stehen und das zwischen Rauten und Titel des Headings eine Leerstelle ist, da sie sonst nicht funktionieren. Leerraum zwischen Headings und dem restlichen Text sind auch erwünscht, da sie sonst auch unter Umständen nicht funktionieren.

Hiermit können wir unserem Text Struktur geben!

Tabellen

Tabellen lassen sich auch per Markdown rein textbasiert erstellen, so wie es auch bei den Headings zuvor gemacht wurde

| Spalte 1 | Spalte 2 |
|---|---|
| Spalte 1, Eintrag 1  | Spalte 2, Eintrag 1 |
| Spalte 1, Eintrag 2  | Spalte 2, Eintrag 2 |

wird dargestellt als

Spalte 1 Spalte 2
Spalte 1, Eintrag 1 Spalte 2, Eintrag 1
Spalte 1, Eintrag 2 Spalte 2, Eintrag 2

Listen

Sowohl sortierte Listen mit Zahlen als auch unsortierte Strichlisten werden automatisch in Markdown konvertiert

1. Eins
2. Zwei
3. Drei

- Eins
- Zwei
- Drei

wird zu

  1. Eins
  2. Zwei
  3. Drei
  • Eins
  • Zwei
  • Drei

Codeblöcke

Mit Backticks ` lassen sich Code-Inhalte anzeigen, mit jeweils einem Backtick um ein Wort wird nur das einzelne Wort Inline als Code dargestellt. Wenn man einen Textabschnitt um ein Tab einrückt oder jeweils drei Backticks unter und über einen Block packt, wird der ganze Block als Code dargestellt. Damit wurden auch zuvor die einzelnen Operatoren aufgeführt, bevor sie in ihrer Anwendung gezeigt wurden.


#### **Vorraussetzungsprüfung**

Der Code dazu sah folgenderweise aus:
```
UV <- fb22$nerd
AV <- fb22$neuro

## Linearität
plot(UV, AV, xlab = "Nerdigkeit", ylab = "Neurozität", 
     main= "Zusammenhang zwischen Gewicht und Treibstoffeffizienz"
     , pch =19)
lines(loess.smooth(UV, AV), col = 'blue')

## Varianzhomogenität
# Siehe Plot
par(mfrow = c(2, 2)) #vier Abbildungen gleichzeitig
plot(model)

## Normalverteilung der Residuen
shapiro.test(model[["residuals"]])
# +plot

## Unabhängigkeit der Residuen
# Siehe plot

```

#### **Ergebnis**

Das Ergebnis ist, dass unser Prädiktor nicht signifikant von Null verschieden ist (p-Wert von 0.3406 bei einem Alpha von .05). Die Alternativhypothese wird verworfen und die Nullhypothese bleibt erhalten.

Der Code dazu sah folgenderweise aus:

Horizontale Trennstriche

Mit drei Asterisks *** oder Underscores ___ kann man eine horizontale Linie als Trenner einbauen. Diese müssen dafür für sich alleine in einer eigenen Zeile stehen und sollten auch eine Zeile Abstand zu restlichem Text haben, da sie sonst zu einem Heading mit den vorherstehenden Inhalten kombiniert werden. Man kann dabei auch beliebig oft die Zeichen wiederholen, statt nur drei mal, sofern man den Code selbst auch visuell trennen möchte.

Mit den Strichen lassen sich beispielsweise die Codeabschnitte besser visuell trennen

Interne Links

Interne Links referenzieren Heading in dem Text. Dafür muss man hinter den jeweiligen Heading eine Leerstelle einbauen und per {#Text} eine Referenz für den internen Link einbauen. Nun baut man wie zuvor erklärt einen Link indem man Text in eckige Klammern schließt, und nun danach wieder danach in der runden Klammer #Text, also die Referenz des internen Links aufruft. Damit kann man nun sofort zu dem Heading springen, wenn man auf den erstellten internen Link drückt!

Wir können also an den Anfang eine kleine Auflistung unserere Überschriften machen und direkt erlauben auf diese Aufzurufen, quasi wird ein kleiner Table of Contents erstellt!

Fazit zu Basic Markdown

Markdown selbst bietet eher grundlegende Funktionen um einen formatierten Text darzulegen, aber die wahre Stärke von Markdown liegt in der Erweiterbarkeit des Grundgerüsts von Markdown durch verschiedene Compiler, weitere Programmiersprachen und andere externe Programme und Formate, die Markdown als Grundlage verwenden.

Markdown und andere Formate

Markdown kann unter anderem zum Erstellen von Websites genutzt werden, was auch einst der Grund für die Erfindung von Markdown war. Dabei lassen sich zusätzlich CSS und HTML nutzen, um weitaus komplexere Anpassungen durchzuführen. Ein Großteil von Pandar besteht auch nur aus Markdown und gelegentlichem HTML! Aber auch für Notizen-Apps kann Markdown interessant sein, besonders wenn man etwa Obsidian benutzt, welches grundlegend mit Markdown arbeitet. Auch Projektdokumentationen, Präsentations-Slides und PDF-Dokumente lassen sich mit Markdown anfertigen und um weitere Features erweitern die speziell für diese Formate geeignet sind. Im nächsten Tutorial wird ein Compiler aufgeführt, der viele dieser Funktionen auch umsetzen kann, Pandoc. Dort werden auch weiter Funktionen vorgestellt, die unseren Miniaturbericht noch besser gestalten können.

RMarkdown hingegen dient spezifisch der visuellen und schriftlichen Dokumentation von Analysen in R und wird in dem letzten Tutorial dieser Reihe besprochen, da es alle vorigen Elemente von Markdown, Pandoc und R kombiniert und die finale Version des Berichts abbilden wird.
