Einführung

GitLab ist ein Versions Kontroll System oder Version Control System (VCS), das vollständig auf Git basiert. Hauptaufgabe von VCSs ist die transparente Änderung und Dokumentation von Dateien. Man soll sie zu jeder Zeit nachvollziehen können. Das Gute an GitLab ist, dass mehrere Personen zeitgleich an einem Projekt mitwirken können und Änderungen den anderen direkt ersichtlich sind. Das ganze Konzept ist natürlich auch von einem Open Science Standpunkt aus sehr interessant, da jegliche Änderung an Dateien einsehbar und nachvollziehbar ist, was die Wahrscheinlichkeit von erfolgreichen Täuschungsversuchen sehr minimal hält. Hier erhaltet ihr einen Einblick in die wichtigsten Funktionen von GitLab, die man als Anfänger wissen muss.

Einloggen

Sobald ihr eure Zugangsdaten bekommen habt, könnt ihr euch auf dem LIFOS-Server einloggen.

Wenn ihr euch zum ersten Mal einloggt, müsst ihr noch die Datenschutz- und Nutzungsbedingungen akzeptieren.

Startseite

Nachdem ihr euch eingeloggt habt, kommt ihr zu folgender Startseite:

Dort seht ihr direkt schonmal das Wichtigste: die Projects. Hier werden euch alle Projekte angezeigt, bei denen ihr irgendwie involviert seid. Direkt unter der großen Überschrift findet ihr eine Leiste, die euch einen Überblick über verschiedene Arten von Projekten gibt mit denen ihr persönlich zu tun habt.

Unter Your Projects findet ihr sowohl Projekte, die ihr selbst erstellt habt (“Personal”), aber auch welche an denen ihr Mitwirkende:r seid (“All”). Wenn ihr euch das erste Mal auf GitLab eingeloggt habt, wird vermutlich nichts unter “Personal” stehen, da ihr selbst noch kein Projekt erstellt habt. Falls ihr aber zu einer Gruppe hinzugefügt wurdet, seht ihr alle Projekte dieser Gruppe unter “All”.

Erwähnenswert ist auch noch Explore Projects. Hierunter könnt ihr euch freigegebende Projekte anderer User:innen anschauen. Falls euch eines dieser Projekte gefällt, da ihr es als Insipiration für euer eigenes Projekt nehmen wollt, könnt ihr dieses liken. Dafür geht ihr auf das entsprechende Projekt und drückt auf Star.

Wenn ihr das getan habt und wieder zur Startseite zurückkehrt, indem ihr links oben auf den Goethe-Button klickt, findet ihr dieses Projekt unter Starred Projects wieder. So könnt ihr alle Projekte, die euch gefallen, leichter wiederfinden.

Nachdem ihr euch also schon mit den verschiedenen Arten von Projekten auskennt, ist es an der Zeit euer erstes eigenes Projekt zu erstellen.

Projekt erstellen

Hierfür müsst ihr unter der Seite Projects auf den Button “New project” rechts oben klicken.

Danach wird euch eine Seite mit vier Optionen angezeigt: Create blank project, Create from template, import project und Run CI/CD for external repository.

Hier wählt ihr die Option Create from template aus. Um auf die für euch relevanten Templates zu kommen, müsst ihr in folgender Spalte auf “Instance”. Die “Built-in” Templates sind globale Templates, die jedem:r GitLab-User:in zur Verfügung stehen. Diese könnt ihr euch ein anderes Mal gerne anschauen oder mit einem Testprojekt erkunden.

Nachdem ihr das Template ausgewählt habt, das auf euer Vorhaben am besten zutrifft, müsst ihr diesem einen Namen geben und das Visibility-Level anpassen. Letzteres ist sehr wichtig, da hier festgelegt wird, wer das Projekt und alle zugehörigen Dateien sehen kann. Hier gibt es drei Optionen.

Private

Wenn ihr diese Option auswählt, könnt nur ihr das Projekt sehen oder alle Mitglieder der Gruppe, die an diesem Projekt mitwirkt. Falls ihr vertrauliche Daten habt, die nicht jeder sehen soll, wäre das die beste Option.

Internal

Wählt ihr diese Option, so kann jeder, der auch Zugang zu der LIFOS-Seite hat, euer Projekt sehen. So könnte euer Projekt zum Beispiel unter Explore Projects für andere zu sehen sein.

Public

Hier könnte wirklich die ganze Welt euer Projekt sehen, da man auch von außerhalb der LIFOS-Seite darauf zugreifen kann. Wählt diese Option also nicht oder nur mit Bedacht.

Außerdem könnt ihr optional noch eine Projektbeschreibung hinzufügen. Danach müsst ihr nur noch auf Create Project klicken und ihr habt euer erstes Projekt erstellt. Das ganze sieht dann so aus:

Projekt - Orientierung

Zu allererst ist das README für euch am wichtigsten, hier steht alles wichtige zu dem Template drinnen. Ihr findet es, wenn ihr auf eurer Projektseite einfach nur nach unten scrollt.

Anschließend findet ihr oben verschiedene Ordner, die euch eine Orientierung geben sollen, was ihr alles bneötigen werdet z.B. Präregistrierung, Skripte etc. Diese sind natürlich noch leer und müssen mit euren Dateien gefüllt werden.

Dateien hochladen

Dafür müsst ihr wissen, wie man Dateien überhaupt auf euer Repository hochlädt. Geht dazu auf der Startseite eueres Projektes auf den Plus-Button.

Es öffnet sich eine Liste mit Optionen. Wir haben schon eine Datei namens “Datenauswertung.R”, die wir hochladen wollen. Deswegen klicken wir auf Upload File. Ihr könnt aber auch direkt auf GitLab eine neue Datei erstellen mit New File.

Es öffnet sich ein Fenster, das euc dazu auffordert eure Datei hochzuladen. Das funktioniert ganz einfach mit Drag-And-Drop Prinzip. Anschließend ist es wichtig eine Commit-Message zu schreiben, damit ihr später wisst, was ihr da überhaupt gemacht habt oder eure Gruppenmitglieder sehen können, was das für eine neue Datei ist.

Zum Abschluss klickt ihr auf Upload File. Ihr findet euere Datei jetzt bei der Übersicht eures Projekts wieder:

Falls ihr eine Datei direkt in einem Ordner erstellen wollt, müsst ihr den gwünschten Ordner öffnen und dann diesselbe Prozedur wie oben wiederholen.

Wir haben unsere Datei in keinen Ordner hochgeladen, sondern direkt auf die “Startseite” unseres Projektes. Uns fällt im Nachhinein auf, dass die Datei aber eigentlich doch besser in dem Ordner “Scripts” untergebracht hätte werden müssen. Normalerweise würde man die Datei jetzt verschieben, allerdings funktioniert das in GitLab nicht. Der einzige Weg, die Datei in den Ordner zu bekommen, ist sie zuerst zu löschen und dann wieder hochzuladen.

Dateien löschen

Um eine Datei zu löschen, klickt ihr auf diese und seht anschließend auch direkt den Delete-Button.

GitLab fragt euch dann noch nach einer Commit-Message für diese Aktion, da es wichtig ist alles zu kommentieren, um es nachvollziehbar zu machen.

Wie ihr seht befindet sich “Datenauswertung.R” jetzt im richtigen Ordner.

Ordnerstruktur ändern

Natürlich ist es auch interessant, nicht nur Dateien zu erstellen und hochladen zu können, sondern auch neue Ordner zu erstellen und zu bearbeiten.

Ordner erstellen

Wir erstellen jetzt mal einen Test-Ordner, in dem ihr einfach Sachen ausprobieren könnt. Dafür gehen wir auf die Startseite unseres Projektes und auf das “+”:

Anschließend öffnet sich eine Toggle-Down Liste bei der wir auf New Directory klicken müssen.

Diesem Ordner gebt ihr nun einen Namen (z.B. “Test-Ordner”) und eine passende Commit-Message.

Anschließend müsst ihr auf Create Directory gehen und ihr habt einen neuen Ordner erstellt! Diesen findet ihr auf der Startseite eures Projekts.

Ordner löschen

Jetzt wollt ihr den Testordner aber wieder löschen, weil ihr gemerkt habt, dass er überflüssig geworden ist. Dies funktioniert in GitLab nur, wenn alle Dateien aus dem Ordner gelöscht werden. In unserem Ordner befindet sich gerade nur die “.gitkeep”-Datei, aber diese reicht schon aus, um den Ordner weiter existieren zu lassen.

Ihr klickt auf diese Datei und müsst anschließend nur noch auf den Delete-Button gehen.

Nachdem ihr die letzte Datei in eurem Ordner gelöscht habt (in diesem Fall “.gitkeep”), verschwindet der Ordner automatisch aus eurem Projekt.

Profil bearbeiten

Zu guter letzt schauen wir uns noch an, wie ihr euer Profil bearbeiten könnt. Dafür müsst ihr oben rechts bei eurem Avatar auf Edit profile gehen.

Hier könnt ihr unter anderem euer Profilbild, euren Status oder sogar eure Zeitzone ändern. Was eventuell wichtiger ist, sind die Haupteinstellungen (die Main Settings). Hier kann man eine Vielzahl an Dingen ändern, die nicht nur eurer Profil betreffen, sondern auch eure Person. So könnt ihr eure bevorzugten Pronomen eingeben oder auch wie man euren Namen ausspricht.

Außerdem kann man diverse Profile verlinken (LinkedIn, Twitter, Skype etc.), sowie eine Bio verfassen.

Anschließend könnt ihr entscheiden, ob ihr euer Profil privat machen wollt oder nicht. Falls ihr euch für ein privates Profil entscheidet, wird folgendes für andere nicht mehr sichtbar:

  • Datum, wann euer Account entstanden ist
  • Tabs für Aktiviäten, Gruppen, beigetragende Projekte, persönliche Projekte, favorisierte Projekte oder Snippets.

Zuletzt klickt ihr noch auf Update profile settings, damit eure Änderungen auch gespeichert werden.
